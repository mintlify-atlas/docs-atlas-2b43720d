Choosing a DNS provider comes down to two questions: what protocol you want to use, and how much filtering you want applied to your queries. This page summarizes every provider available in the encrypted-dns project so you can make an informed choice before downloading a profile.

## Protocols: DoH vs DoT

**DNS over HTTPS (DoH)** tunnels queries inside standard HTTPS traffic on port 443. It is the most widely supported option and the hardest for networks to block or identify. **DNS over TLS (DoT)** uses a dedicated encrypted connection on port 853. DoT is easier for network operators to detect and block, but it cleanly separates DNS from general HTTPS traffic.

<Info>
  Most providers offer DoH. Not all providers offer DoT. If you are on a restrictive network, prefer DoH profiles.
</Info>

## Filtering types

The project classifies providers along a single axis called **censorship**. A provider with `censorship: true` will not return the true IP address for some hostnames — it withholds or redirects records to enforce a policy. That policy can be useful (blocking malware) or a matter of preference (blocking ads). Four categories are used in this documentation:

- **No filtering** — pure DNS resolution, no records withheld.
- **Ad-blocking** — ads, trackers, and sometimes malware domains are blocked.
- **Family protection** — adult content and malware are blocked in addition to the above.
- **Security filtering** — malware and phishing domains are blocked, but adult content is not.

## Signed vs unsigned profiles

Each `.mobileconfig` file is available in a signed and unsigned form. Signed profiles carry a cryptographic signature so iOS and macOS can verify they have not been tampered with. The download links on every page in this section point to the **unsigned** profiles hosted directly on GitHub.

<Note>
  Signed profiles are in the `signed/` directory of the repository and have the same filename as their unsigned counterparts. You can also build your own signed copies — see [signing profiles](/contributing/signing-profiles) for instructions.
</Note>
